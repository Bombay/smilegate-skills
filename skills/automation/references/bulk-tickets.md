# 시나리오 3: 티켓 한번에 만들기

## 개요
- **입력**: 자연어로 작성된 요구사항/할 일 목록
- **출력**: Jira에 생성된 여러 개의 티켓
- **필요 도구**: Jira
- **소요 시간**: 약 2~3분

## 실행 절차

### Step 1: 프로젝트 선택

사용자의 Jira 프로젝트 **목록을 먼저 조회**한다:
```
mcp__jira__get_all_projects()
```

목록을 보여주고 선택받는다:
```
📂 어떤 프로젝트에 티켓을 만들까요?

1. 웹 리뉴얼
2. 모바일 앱
3. 백오피스

번호로 골라주세요!
```

AskUserQuestion으로 선택을 받는다.

> 사용자에게 보이는 메시지:
> ```
> ⏳ Jira 프로젝트 목록을 가져오고 있어요...
> ```

### Step 2: 요구사항 입력

AskUserQuestion으로 요구사항을 받는다:
```
📝 만들고 싶은 티켓들을 자유롭게 적어주세요!

형식은 상관없어요. 이런 식으로 적어주시면 돼요:
• "로그인 페이지 수정, 비밀번호 찾기 추가, 마이페이지 변경"
• "1. 가입 플로우 개선 2. 이메일 인증 추가"
• 또는 그냥 편하게 설명해주세요!
```

### Step 3: 요구사항 분석 및 티켓 분해

사용자의 입력을 분석하여 개별 티켓으로 분해한다:

```
📋 이렇게 티켓을 만들어볼게요!

1. 로그인 페이지 디자인 수정
   → 로그인 페이지의 UI/UX를 최신 디자인 가이드에 맞게 수정

2. 비밀번호 찾기 기능 추가
   → 이메일 기반 비밀번호 재설정 기능 구현

3. 마이페이지 레이아웃 변경
   → 마이페이지 레이아웃을 모바일 반응형으로 변경

이대로 만들까요? 수정할 부분이 있으면 말씀해주세요!
```

AskUserQuestion으로 확인을 받는다.

> 사용자에게 보이는 메시지:
> ```
> 📝 요구사항을 분석하고 있어요...
> ```

### Step 4: 티켓 생성

확인을 받으면 각 티켓을 순차적으로 생성한다:
```
mcp__jira__create_issue(
  project_key="{선택된 프로젝트 키}",
  summary="{티켓 제목}",
  description="{티켓 설명}",
  issue_type="Task"
)
```

> 사용자에게 보이는 메시지 (각 티켓 생성 시):
> ```
> 🎫 티켓 만드는 중... (1/3)
> 🎫 티켓 만드는 중... (2/3)
> 🎫 티켓 만드는 중... (3/3)
> ```

> **주의**: 각 티켓 생성 후 결과를 확인하고, 실패 시 사용자 친화적으로 안내

### Step 5: 결과 안내

```
✨ 완료!

🎫 생성된 티켓
┌──────────────────────────────────────┐
│ WEB-101  로그인 페이지 디자인 수정      │
│ WEB-102  비밀번호 찾기 기능 추가        │
│ WEB-103  마이페이지 레이아웃 변경       │
└──────────────────────────────────────┘

3개의 티켓을 한번에 만들었어요!
```

## 스킬 생성 시 템플릿

**자연어로 작성한다 (MCP 함수명 사용 금지):**

```yaml
---
name: 티켓생성
description: >
  요구사항을 분석하여 Jira 티켓을 한번에 생성한다.
triggers:
  - "티켓생성"
  - "티켓 생성"
  - "티켓 만들기"
  - "일괄 티켓"
---

# 티켓 일괄 생성

요구사항을 분석하여 Jira 티켓을 한번에 생성한다.

## 실행 절차
1. 요구사항을 개별 작업 단위로 분해
2. 각 작업의 제목과 설명을 구조화
3. 사용자 확인 후 {프로젝트명} 프로젝트에 티켓 일괄 생성
4. 생성된 티켓 목록 출력
```
