# 스킬 생성 가이드

## 개요

사용자가 체험한 자동화 워크플로우를 **skills/ 디렉토리의 독립 스킬**로 저장한다.
이후 사용자는 스킬 이름을 말하기만 하면 같은 자동화를 반복 실행할 수 있다.

## 스킬 파일 구조

### 디렉토리 구조
```
skills/
├── automation/      ← 이 스킬 (자동화 만들기)
├── connector/       ← 도구 연결
└── {스킬명}/        ← 사용자가 만든 스킬
    └── SKILL.md
```

### SKILL.md 형식

```yaml
---
name: {스킬명}
description: >
  {한 줄 설명 — 무엇을 하는 스킬인지}
triggers:
  - "{스킬명}"
  - "{스킬명의 다른 표현}"
  - "{영어 표현}"
---

# {스킬 제목}

{자세한 설명}

## 실행 절차

1. {Step 1 — 자연어로 무엇을 하는지}
2. {Step 2}
3. {Step 3}
...
```

### 이름 규칙
- **한글** 사용 권장 (비개발자 친화): `주간보고`, `회의록`, `티켓생성`
- 짧고 직관적으로: 2~4글자
- 디렉토리명 = name 필드 값

### triggers 규칙
- 스킬명 그대로: `"주간보고"`
- 자연어 변형: `"주간 보고"`, `"주간보고 해줘"`
- 영어 (선택): `"weekly report"`
- 최소 2개 이상 등록

### 중요: MCP 함수명 사용 금지

스킬 설명에 `mcp__jira__myself()` 같은 기술적 함수명을 쓰지 않는다.
Claude는 자연어 설명만으로도 올바른 도구를 찾아 실행할 수 있다.

**나쁜 예시** (기술 용어 노출):
```yaml
---
name: 주간보고
---
# 주간보고
1. `mcp__jira__myself()`로 내 정보 확인
2. JQL `assignee = currentUser() AND updated >= startOfWeek()`로 티켓 조회
```

**좋은 예시** (자연어):
```yaml
---
name: 주간보고
description: >
  Jira에서 이번 주 내 티켓을 조회하여 주간 업무 리포트를 위키에 자동 작성한다.
triggers:
  - "주간보고"
  - "주간 보고"
  - "weekly report"
---

# 주간 업무 리포트

Jira에서 이번 주 내 티켓을 조회하여 주간 업무 리포트를 위키에 자동 작성한다.

## 실행 절차

1. Jira에서 내 정보 확인
2. 이번 주 업데이트된 내 티켓 전체 조회
3. 상태별(완료/진행중/대기)로 분류하여 보고서 작성
4. 주요 성과, 다음 주 전망, 리스크 요약
5. {스페이스명} 위키에 "주간 업무 리포트 — {이름} ({날짜})" 페이지 생성
```

## 스킬 파일 생성 절차

### 1. 디렉토리 생성

`skills/` 디렉토리 아래에 스킬명으로 새 폴더를 만든다:
```
skills/{스킬명}/SKILL.md
```

### 2. SKILL.md 작성

위의 좋은 예시 형식을 따라 YAML frontmatter + 마크다운 본문을 작성한다.

### 3. 생성 후 안내

스킬 생성 후 사용자에게 보여줄 메시지:

```
✅ 스킬이 만들어졌어요!

🏷️ 이름: 주간보고
📄 사용법: "주간보고" 또는 "주간보고 해줘"라고 말하면
방금과 같은 자동화가 실행돼요!

💡 나중에 수정하고 싶으면 "주간보고 스킬 수정해줘"라고 말씀해주세요.
💡 다른 자동화도 같은 방법으로 스킬로 만들 수 있어요!
```

## 스킬 충돌 처리

### 이름이 겹치는 경우

기존 skills/ 디렉토리를 확인하여 동일한 이름의 스킬이 있는지 확인한다.

겹치는 경우:
```
"{스킬명}" 스킬이 이미 있어요!

① 기존 스킬을 업데이트하기 (새 버전으로 수정)
② 다른 이름으로 만들기
③ 기존 스킬을 먼저 확인하기

어떻게 할까요?
```

### 기존 스킬 수정

사용자가 "주간보고 스킬 수정해줘"라고 요청하면:
1. skills/ 디렉토리에서 해당 스킬을 찾는다
2. 현재 SKILL.md 내용을 보여준다
3. 수정 사항을 확인한다
4. SKILL.md를 업데이트한다 (Edit으로 수정)

### PPT 출력 스킬

```yaml
---
name: 현황발표
description: >
  Jira 프로젝트 현황을 PPT 발표자료로 자동 생성한다.
triggers:
  - "현황발표"
  - "현황 PPT"
  - "발표자료"
  - "프로젝트 현황 발표"
---

# 프로젝트 현황 발표자료

Jira 프로젝트 현황을 PPT 발표자료로 자동 생성한다.

## 실행 절차

1. Jira에서 프로젝트 선택
2. 전체 이슈 현황 조회 (완료/진행중/대기)
3. 핵심 지표 정리 (완료율, 주요 마일스톤, 리스크)
4. PPT 파일 생성
   - 표지 슬라이드: 프로젝트명, 날짜, 작성자
   - 현황 요약 슬라이드: 완료율, 이슈 분포
   - 상세 이슈 목록 슬라이드: 주요 완료/진행중 항목
   - 리스크 및 다음 단계 슬라이드
5. 파일 저장 및 경로 안내
```

### Excel 출력 스킬

```yaml
---
name: 티켓정리
description: >
  Jira 티켓 목록을 엑셀 파일로 정리하여 내보낸다.
triggers:
  - "티켓정리"
  - "티켓 엑셀"
  - "이슈 엑셀"
  - "티켓 정리"
---

# 티켓 엑셀 정리

Jira 티켓 목록을 엑셀 파일로 정리하여 내보낸다.

## 실행 절차

1. Jira에서 프로젝트 선택
2. 기간/상태 필터 확인
3. 티켓 목록 조회 및 정리
4. Excel 파일 생성
   - 헤더 행: 티켓번호, 제목, 상태, 담당자, 우선순위, 업데이트일
   - 조건부 서식: 상태별 색상 구분
   - 필터 설정
5. 파일 저장 및 경로 안내
```

### 복합 출력 스킬 (데이터 → PPT + Excel)

```yaml
---
name: 스프린트리뷰
description: >
  Jira 스프린트 이슈와 BISKIT 데이터를 종합하여 리뷰 자료를 생성한다.
triggers:
  - "스프린트리뷰"
  - "스프린트 리뷰"
  - "sprint review"
---

# 스프린트 종합 리뷰

Jira 스프린트 이슈와 BISKIT 데이터를 종합하여 리뷰 자료를 생성한다.

## 실행 절차

1. Jira에서 현재/최근 스프린트 정보 확인
2. 스프린트 내 모든 이슈 조회
3. 완료/미완료로 분류하고 요약
4. BISKIT에서 관련 데이터 조회
5. 결과물 형태 선택 (화면 표시 / PPT / Excel / 위키)
6. 선택한 형태로 종합 리포트 생성
```

## 주의사항

- **기존 스킬을 절대 삭제하지 않는다** (수정만 가능)
- 스킬 설명에 하드코딩된 값(스페이스명, 프로젝트명 등)은 사용자가 확인한 값만 사용
- 민감한 정보(토큰, 비밀번호)를 스킬에 포함하지 않는다
- **사용자에게 "SKILL.md", "YAML", "skills/ 디렉토리" 등 기술 용어를 쓰지 않는다**
- PPT/Excel 생성 시 필요한 Python 패키지(python-pptx, openpyxl)는 자동으로 설치/사용한다
