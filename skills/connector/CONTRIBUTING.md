# 신규 커넥터 추가 가이드

이 문서는 Smilegate Connector 스킬에 새로운 서비스를 추가하는 방법을 설명한다.

## 추가 절차 체크리스트

새로운 커넥터를 추가할 때 아래 항목을 순서대로 진행한다:

1. `references/` 에 서비스명.md 파일 생성
2. `SKILL.md`의 연결 대상 테이블에 행 추가
3. `SKILL.md`의 Phase 0 진단에 확인 항목 추가
4. `SKILL.md`의 서비스 선택 옵션에 추가
5. `SKILL.md`의 분기 흐름 업데이트
6. 해당 Phase에 참조 링크 추가
7. Phase 5 완료 리포트에 사용 예시 추가
8. YAML frontmatter의 triggers에 키워드 추가

## 인증 필요 서비스 vs 인증 불필요 서비스

### 인증 필요 서비스 (Jira, Confluence, BISKIT 등)

- Phase 3에서 함께 처리한다
- 토큰 발급 → 토큰 입력 → 설정 파일 저장 → 재시작 → 연결 테스트 흐름을 따른다
- 가이드 파일에 토큰 발급 방법, 토큰 입력 AskUserQuestion, MCP 설정 JSON, 연결 테스트 방법을 포함한다
- Phase 0 진단에서 ✅/⚠️/❌ 세 가지 상태를 모두 처리한다

### 인증 불필요 서비스 (API Docs 등)

- Phase 2에서 처리한다 (또는 별도 Phase 추가)
- `claude mcp add` 명령으로 간단히 설치한다
- 가이드 파일에 설치 명령, 재시작 안내, 연결 테스트 방법을 포함한다
- Phase 0 진단에서 ✅/❌ 두 가지 상태만 처리한다 (인증이 없으므로 ⚠️ 상태 없음)

## 가이드 파일 템플릿

새 서비스의 `references/{서비스명}.md` 파일은 아래 구조를 따른다:

```markdown
# {서비스명} 설정 가이드

{서비스 설명 1-2줄}

> ⚠️ {네트워크 요구사항이 있으면 여기에 표시}

## {인증 방법} (인증 필요 서비스인 경우)

{토큰/키 발급 방법을 단계별로 안내}

## 토큰 입력 (인증 필요 서비스인 경우)

{AskUserQuestion 스펙}

## 설치 명령 (인증 불필요 서비스인 경우)

{claude mcp add 명령}

## MCP 설정

{~/.claude.json에 추가할 JSON 설정}

## 연결 테스트

{연결 테스트 방법과 성공/실패 시 안내}

## 트러블슈팅

{서비스별 트러블슈팅 체크리스트. 진전 없으면 troubleshoot.md 참조 안내}

## 관련 URL

| 항목 | URL |
|------|-----|
| {서비스} MCP 서버 | {URL} |
| {기타 관련 링크} | {URL} |
```

## 네이밍 규칙

- 파일명: 서비스명 소문자 (예: `apidocs.md`, `biskit.md`, `slack.md`)
- MCP 서버 키: 서비스명 소문자, 하이픈 사용 (예: `biskit-report-mcp`)
- YAML triggers: 사용자가 자연스럽게 부를 수 있는 모든 변형 포함 (예: "biskit 연결", "비스킷 연결")
